1. EXECUTIVE SUMMARY
--------------------
Screen Entry is a professional casting platform built for the Telugu film
industry. It connects actors with casting directors, production houses, and
recruiters. The platform is currently in private beta, accessible only via
invite code.

The application is a full-stack web app built on Next.js 15 with Supabase as
the backend. All auth flows are server-side. The frontend follows a cinematic
dark design system with gold and red brand colors.

6. AUTHENTICATION ARCHITECTURE
--------------------------------
All auth is server-side. The browser never handles raw session tokens.

FLOWS
-----
Flow              Endpoint                 Description
---------------------------------------------------------------------------
Sign up           POST /api/auth/signup    Validates invite code, signUp(), sets cookie
Sign in           POST /api/auth/login     signInWithPassword() server-side, sets cookie
Sign out          POST /api/auth/signout   signOut() server-side, clears cookie
Email confirm     GET /auth/callback?code  exchangeCodeForSession(), redirect /dashboard
Password reset    GET /reset-password?code exchangeCodeForSession(), updateUser()
Session refresh   middleware.ts            Runs on every request, refreshes cookie

INVITE-ONLY ACCESS
------------------
Signup requires a valid invite code stored in the INVITE_CODE environment
variable on Vercel (server-side only - never exposed to the browser).
Invalid codes return a 403 error before signUp is called.

KEY RULES
---------
- All post-auth redirects use window.location.href, never router.push()
  Reason: ensures server reads the fresh session cookie on redirect
- Supabase client in React components always created with:
  useRef(createClient()).current
  Reason: prevents re-creation on every render which causes auth loops


7. STORAGE ARCHITECTURE
------------------------
Supabase Storage Bucket: "Screen Entry" (public)

Path                          Content           Upload Rule
---------------------------------------------------------------------------
{userId}/avatar.{ext}         Profile photo     Upsert - replaces previous
{userId}/{timestamp}.{ext}    Portfolio photo   Max 5 per user, max 5MB each

Avatar URLs include ?t={timestamp} cache-bust param to force browser refresh.
